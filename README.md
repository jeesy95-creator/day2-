# 💪 InBody 회원 분석 대시보드

InBody 측정결과와 회원관리 데이터를 업로드하면 자동으로 통합·분석해주는 Streamlit 기반 인터랙티브 대시보드입니다.

---

## 주요 기능

### 기본 분석
| 기능 | 설명 |
|------|------|
| **파일 업로드 & 자동 머지** | 측정결과 + 회원관리 두 파일을 `회원ID` 기준으로 자동 병합 |
| **KPI 카드** | 분석 회원 수 / 평균 BMI / 평균 체지방률 / 평균 만족도 한눈에 확인 |
| **이용센터별 체지방·근육량** | 센터별 평균 체지방률(바 차트)과 골격근량(라인 차트) 이중 축 비교 |
| **체지방 vs 골격근 산점도** | 성별 구분 색상으로 개인 분포 시각화 |
| **트레이너별 만족도** | 담당 트레이너별 평균 만족도 랭킹 바 차트 |
| **목표 체중 달성 현황** | 현재 체중 − 목표 체중 차이를 개인별로 시각화 (초과/달성 색상 구분) |
| **내장지방 레벨 분포** | 레벨별 인원 수 + 위험도(정상/경계/위험) 색상 표시 |
| **프로그램별 현황** | 프로그램별 회원 수 및 평균 주운동빈도 복합 차트 |

### 심층 분석
| 기능 | 설명 |
|------|------|
| **BMI 분포 히스토그램** | 저체중 / 정상 / 과체중 / 비만 구간별 인원 분포 (색상 구분) |
| **나이대별 체성분 비교** | 20대~60대+ 그룹별 평균 체지방률 & 골격근량 이중 축 차트 |
| **월별 체성분 트렌드** | 측정일 기반 월별 평균 추이 (체중 / 체지방률 / 골격근량 / BMI 선택) |
| **이탈 위험 회원 목록** | 운동빈도·만족도·회원기간 기준 위험 점수 산출 → 고위험/주의 자동 분류 |

### 영양 가이드 (USDA FoodData Central API)
| 기능 | 설명 |
|------|------|
| **회원 평균 권장 섭취량** | Mifflin-St Jeor 공식으로 유지 칼로리·목표 칼로리·권장 단백질 계산 |
| **헬스 추천 식품 비교** | 닭가슴살·고구마·계란·그릭요거트·현미밥 영양 정보 차트 + 테이블 |

### 필터 & 탐색
| 기능 | 설명 |
|------|------|
| **다이나믹 필터** | 이용센터 / 프로그램 / 성별 멀티셀렉트 필터로 전체 차트 동시 반영 |
| **통합 데이터 탐색** | 머지된 전체 데이터 테이블 (컬럼 선택 / 전체 보기) |

## 이탈 위험 회원 판단 기준

| 항목 | 위험 조건 | 점수 |
|------|-----------|------|
| 주운동빈도 | 주 2회 이하 | +1 |
| 만족도 | 3.0점 이하 | +1 |
| 회원기간 | 3개월 이하 | +1 |

- **2점**: ⚠️ 주의 — 모니터링 권장
- **3점**: 🔴 고위험 — 즉각 관리 필요

## 입력 파일 형식

사이드바에서 두 파일을 모두 업로드해야 대시보드가 활성화됩니다.

### `inbody_측정결과.xlsx`

| 컬럼명 | 설명 |
|--------|------|
| 회원ID | 고유 식별자 (머지 키) |
| 이름 | 회원 이름 |
| 성별 | 남 / 여 |
| 나이 | 만 나이 |
| 키(cm) | 신장 |
| 체중(kg) | 현재 체중 |
| BMI | 체질량지수 |
| 체지방률(%) | 체지방 비율 |
| 골격근량(kg) | 골격근 무게 |
| 내장지방레벨 | 내장지방 등급 (숫자) |
| 측정일 | 측정 날짜 |

### `inbody_회원관리.xlsx`

| 컬럼명 | 설명 |
|--------|------|
| 회원ID | 고유 식별자 (머지 키) |
| 이용센터 | 등록된 지점 |
| 프로그램 | 수강 프로그램 |
| 담당트레이너 | 담당 트레이너 이름 |
| 목표체중(kg) | 목표 체중 |
| 만족도(5점) | 서비스 만족도 |
| 주운동빈도(회) | 주당 운동 횟수 |
| 회원기간(개월) | 등록 유지 개월 수 |
| 등록일 | 최초 등록 날짜 |

## 실행 방법

```bash
# 의존성 설치
pip install -r requirements.txt

# 앱 실행
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 로 접속한 뒤, 사이드바에서 두 엑셀 파일을 업로드하면 분석이 시작됩니다.

### USDA 영양 API (선택)

사이드바 **STEP 3** 항목에 [USDA FoodData Central](https://fdc.nal.usda.gov/api-guide.html) API Key를 입력하면 더 높은 요청 한도로 사용할 수 있습니다. 키를 비워두면 `DEMO_KEY`(시간당 30회 / 일 1,000회)로 자동 동작합니다.
